## 2. AI 输出
### （1）项目目录结构分析
- `.vscode`：VSCode编辑器配置文件夹，存放编辑器个性化设置、插件配置
- `docs`：项目文档目录，存放实验文档、说明文档，本次AI记录文档放置在 docs/ai 下
- `node_modules`：项目所有依赖包安装目录，npm/pnpm自动生成
- `public`：静态资源根目录，存放不会被Vite编译的图片、html、静态资源
- `scripts`：自定义脚本文件，用于打包、部署、环境处理等脚本逻辑
- `src`：项目源代码核心目录
  - api：后端接口请求封装，统一管理HTTP请求
  - assets：样式、图片、字体等编译型静态资源
  - components：全局公共复用组件
  - router：路由配置文件夹，存放路由规则、路由守卫
  - store：全局状态管理（Pinia）
  - utils：通用工具函数封装
  - views：页面级业务视图组件
  - App.vue：根组件
  - main.ts：项目入口启动文件
- `.gitattributes / .gitignore`：Git配置文件，忽略不需要提交的文件
- `index.html`：页面入口HTML模板
- `package.json`：项目依赖、脚本命令、项目描述配置文件
- `tsconfig.json`：TypeScript类型编译配置
- `vite.config.ts`：Vite构建工具配置文件

### （2）main.ts 解释
main.ts 是整个Vue单页应用**程序入口文件**，程序最先执行该文件：
1. 导入 createApp 创建Vue应用实例；
2. 引入全局样式、路由实例、全局状态管理；
3. 使用 `.use()` 注册路由、状态库、第三方插件；
4. 挂载根组件 App.vue；
5. 通过 mount('#app') 将整个Vue应用挂载到 index.html 的DOM节点上，完成项目初始化启动。

### （3）router 路由作用
1. 实现SPA单页面无刷新跳转，替换传统a标签页面刷新跳转；
2. 匹配URL地址与页面组件，地址变化自动渲染对应views页面；
3. 配置全局/局部路由守卫，实现登录鉴权、访问权限拦截、页面跳转控制；
4. 支持路由参数传递，页面之间传参；
5. 支持hash、history两种地址模式，优化浏览器地址栏展示。

### （4）项目技术栈
- 前端框架：Vue 3
- 类型语言：TypeScript
- 构建工具：Vite
- 路由管理：Vue Router 4.x
- 状态管理：Pinia
- 代码规范：ESLint + Prettier
- 网络请求：Axios
- 版本控制：Git + GitHub

## 3. 自己的理解
1. 该项目采用企业标准Vue工程分层结构，目录职责划分清晰，便于多人协作开发与后期维护迭代；
2. main.ts 是项目启动总入口，所有全局插件、全局配置统一在此初始化，是整个项目运行起点；
3. 路由是单页应用核心，实现页面跳转与权限控制，是前后端分离项目必备模块；
4. Vue3 + TS + Vite 是目前前端主流企业技术栈，启动速度快、类型约束严谨、维护性更强。
